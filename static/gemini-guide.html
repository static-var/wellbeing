Gemini API key reference

This page explains how to obtain a Gemini API key, what the preset models are generally suited for, and what it means for your data when you choose to use one.

Understand before you enable

When you add a personal Gemini key, Wellbeing sends your chat messages directly to Google's Gemini API using that key. From that point Google — not Wellbeing — receives, processes, and may retain the content of those requests.

Google's own API terms, privacy policy, and data-retention rules govern what happens to that data. We have no control over Google's logging, storage, review, or training practices, and we are not responsible for how Google processes what you send through its service.

Review Google's current policies yourself before enabling a key. Those policies can change at any time.

How to get a key

Open Google AI Studio

Go to aistudio.google.com and sign in with the Google account you want associated with the key.

Create an API key

Use the API-key screen to generate a new key, then copy it. It will typically start with AIza….

Paste it into Wellbeing

In onboarding or settings, check "Use my own Gemini API key", choose a model preset, paste the key, and save.

Preset models

These are the models available in the preset dropdown. The descriptions below are general guidance, not guarantees — actual behavior depends on Google's current model versions.

Preset General character
gemini-2.5-flash Fast and broadly capable. Typically the most practical default — reasonable quality with lower quota usage.
gemini-2.5-pro Tends toward longer, more nuanced responses. Uses more quota per request and may respond more slowly.
gemini-2.0-flash A lighter, older model. Useful if you want minimal quota consumption and don't need the latest capabilities.
